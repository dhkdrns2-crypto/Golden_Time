# Golden Time Project 보안 가이드

본 문서는 Golden Time Project(프론트엔드 Vue, 백엔드 Spring Boot, OCR 서버 FastAPI/Python)의 개발/운영 보안 기준을 정의한다.

## 1. 목적 및 적용 범위

- 목적: 개인정보 및 서비스 자산을 보호하고, 웹/시스템 취약점을 사전 예방한다.
- 범위: 프론트엔드, 백엔드 API, DB, 파일 업로드, OCR 서버, 배포 인프라, 운영 절차.
- 원칙:
  - 최소 권한(Least Privilege)
  - 안전한 기본값(Secure by Default)
  - 방어 심층화(Defense in Depth)
  - 로그 기반 탐지 및 신속 대응

## 2. 개인정보 및 민감정보 보호 기준

### 2.1 저장 데이터 보호

- 비밀번호:
  - 평문 저장 금지
  - BCrypt/Argon2 등 단방향 해시 사용 (권장: BCrypt work factor 상향 관리)
- 개인정보(아이디, 이름, 이메일, 전화번호, 주소):
  - 원칙적으로 DB 컬럼 암호화 적용
  - 클라우드 사용 시 KMS 우선 사용
  - KMS 미사용 시 AES-256 이상(예: AES-256-GCM) 적용
- 로그/에러에 개인정보 출력 금지
- 백업 파일/덤프 파일도 동일한 암호화 및 접근통제 적용

### 2.2 전송 데이터 보호

- 외부/내부 API 통신은 TLS(HTTPS) 강제
- 세션 쿠키는 Secure, HttpOnly, SameSite 정책 적용
- 운영 환경에서 평문 HTTP 차단

### 2.3 키/비밀정보 관리

- DB 비밀번호, API 토큰, 암호화 키를 소스코드에 하드코딩 금지
- `.env`, CI Secret, KMS/Secret Manager 사용
- 키 순환(rotate) 정책 수립 및 정기 수행

## 3. 인증/인가 정책

- 인증:
  - 서버 세션 기반 인증 사용
  - 비정상 로그인 시도 탐지 및 제한(계정 잠금/지연/캡차 등)
- 인가:
  - 역할 기반 접근제어(RBAC) 적용
  - 관리자 API는 `ADMIN` 권한만 허용
  - 본인 자원 접근은 소유권 검사 필수(차량/프로필 등)
- 세션:
  - 로그인/로그아웃 시 세션 재생성 및 무효화
  - 유휴/절대 만료시간 정책 운영

## 4. 웹 보안 기준 (OWASP Top 10 중심)

### 4.1 입력 검증 및 출력 인코딩

- 서버측 검증 필수(클라이언트 검증은 보조 수단)
- 화이트리스트 기반 검증 적용
- HTML 출력 시 컨텍스트별 인코딩 적용

### 4.2 SQL Injection 방지

- JPA/Prepared Statement 사용
- 동적 SQL 문자열 결합 금지
- DB 계정 권한 최소화

### 4.3 XSS 방지

- CSP(Content-Security-Policy) 유지/강화
- 인라인 스크립트 최소화
- 사용자 입력 렌더링 시 escaping 적용

### 4.4 CSRF 방지

- 세션 기반 요청은 CSRF 토큰 기본 적용
- 예외 엔드포인트는 최소화하고 문서화/리뷰 필수

### 4.5 파일 업로드 보안

- 허용 확장자/Content-Type/파일 시그니처 검증
- 업로드 크기 제한
- 실행 파일 업로드 차단
- 저장 경로 분리(웹루트 외부 권장), 랜덤 파일명 사용
- 악성코드 검사(백신/샌드박스) 연계 권장

### 4.6 서버 정보 노출 방지

- 상세 스택트레이스/프레임워크 버전/시스템 경로 외부 노출 금지
- 오류 메시지는 사용자 친화적 일반 문구로 통일

### 4.7 HTTP 보안 헤더

- CSP, X-Content-Type-Options, Referrer-Policy, Frame-Options 유지
- HSTS(운영 HTTPS 환경) 적용 권장

## 5. Python OCR 서버 보안 기준

- FastAPI 엔드포인트(`/ocr`) 입력 파일 검증 강화:
  - 확장자/크기/길이 제한
  - 비디오 파싱 타임아웃 설정
- 임시 파일 처리:
  - 예외 발생 여부와 관계없이 삭제 보장
  - 디스크 사용량 모니터링 및 정리 정책
- 모델 파일/가중치 접근권한 최소화
- OCR 서버도 인증된 내부망 또는 API Gateway 뒤에서만 접근 허용

## 7. 취약점 대응 정책

- 취약점 등급 분류:
  - Critical: 24시간 이내 대응
  - High: 3영업일 이내
  - Medium: 10영업일 이내
  - Low: 차기 배포 내 반영
- 계약 종료 이후 발견된 중대 취약점도 조치 원칙 유지

## 8. 코드/설정 표준

- Java(Spring):
  - SecurityConfig 변경 시 보안 리뷰 1인 이상 필수
  - 권한 검사/예외 엔드포인트는 주석으로 이유 기록
- JavaScript(Vue):
  - 인증 상태 및 권한 분기 로직 중앙화
  - 사용자 입력을 HTML로 직접 주입 금지
- Python(FastAPI):
  - 요청 모델 검증(Pydantic) 적극 사용
  - 파일 처리 예외 루틴 표준화

- 정책 예외는 사유/기간/승인자를 기록
